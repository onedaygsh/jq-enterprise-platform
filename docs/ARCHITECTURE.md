# 架构设计文档

## 设计理念

基于华为管理方法论核心思想：

- **封闭式思维**：每个业务域自成闭环，域间通过标准化接口通信
- **双系统理论**：战略层（OKR/目标）与执行层（工单/报工）双向对齐
- **知行合一**：从集团目标到产线执行，逐级分解、实时反馈
- **统一语言**：主数据统一编码，消除多系统口径差异
- **IPD 集成产品开发**：从需求到产品到项目的全生命周期管理

## 系统分层

```
┌────────────────────────────────────────────┐
│              集团驾驶舱（Dashboard）          │
│    四级穿透：集团 → 工厂 → 车间 → 产线        │
├────────────────────────────────────────────┤
│  战略管理  │  经营管理  │  财务管理  │ 人才发展 │
├────────────────────────────────────────────┤
│         主数据中心（统一编码底座）              │
│  组织 │ 工厂 │ 车间 │ 产线 │ 物料 │ BOM │ 设备  │
├────────────────────────────────────────────┤
│   制造执行(MES)  │ 供应链(SCM) │ 产品研发(IPD)  │
│ 工单/报工/采集   │ 采购/库存    │ 需求/项目/质量  │
├────────────────────────────────────────────┤
│             平台底座（Platform）              │
│   认证授权 │ API 网关 │ 日志审计 │ 消息通知     │
└────────────────────────────────────────────┘
```

## 数据模型关系

```
Organization (集团/事业部)
    └── Factory (工厂)
         └── Workshop (车间)
              └── ProductionLine (产线)
                   └── Equipment (设备)

ProductLine (产品线)
    └── Product (产品)
         ├── BOM → Material (物料)
         ├── Requirement (需求)
         └── Project (项目)
              └── Task (任务)

Customer → Order → WorkOrder → ProductionReport
Supplier → PurchaseOrder
Warehouse → Inventory
```

## 事件驱动链路

```
订单创建 → 状态: pending
   ↓ 一键生成工单
工单创建 → 状态: planned → 释放 → 状态: released
   ↓ 一键报工
报工提交 → 更新工单完成数 → 触发订单状态
   ↓ 设备采集
OEE 计算 → 异常检测 → 自动生成报警
```

## API 设计规范

### 路由结构

```
/api/v1/
├── /auth              # 认证（登录/注册/刷新）
├── /group/            # 集团级（蓝图/主数据总览/穿透）
├── /master-data/      # 主数据 CRUD
├── /manufacturing/    # 制造执行
├── /business/         # 经营管理
├── /strategy/         # 战略管理
├── /product/          # 产品研发
├── /finance/          # 财务管理
├── /talent/           # 人才发展
├── /supply-chain/     # 供应链
├── /dashboard/        # 外部数据源（天工 OS）
├── /products/         # 外部产品数据
├── /orders/           # 外部订单数据
└── /suppliers/        # 外部供应商数据
```

### 统一响应格式

```json
// 列表
[...]

// 单条
{ "id": 1, "name": "...", ... }

// 错误
{ "detail": "错误信息" }
```

### 认证方式

所有业务接口需携带 JWT Token：
```
Authorization: Bearer <token>
```

## 安全设计

- **认证**：JWT Token，支持过期刷新
- **授权**：RBAC 角色权限模型
- **数据隔离**：按组织层级过滤
- **接口安全**：CORS 白名单、请求频率限制
- **审计**：操作日志全量记录
